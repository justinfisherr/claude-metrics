# Jazz Taste ML — Claude Guide

This repo holds Justin's personal jazz recommendation ML pipeline: training, prediction, and an interactive dashboard.

## What This Is

A self-improving system that learns Justin's jazz taste from rated tracks and recommends new ones. The training data lives in his Obsidian vault (`~/Documents/remote/Music/Jazz Dataset/training-data.md`), not in this repo.

## Repo Layout

- `train.py` — Reads training data, engineers features, trains Ridge + Random Forest models, outputs `model.joblib` and `dashboard-data.json`
- `predict.py` — Scores candidate tracks against the trained model. Accepts `--json` or `--file` input.
- `model.joblib` — Trained model artifact
- `mood-axes.json` — VAD (valence, arousal, dominance) scores for each mood chip, used by train.py and predict.py
- `dashboard-data.json` — Generated by `train.py`, consumed by the React dashboard
- `versions.json` — Model version manifest
- `versions/` — Major version snapshots
- `src/` — React dashboard (Vite + Chart.js). 33 panel components in `src/components/panels/`
- `dist/` — Build output for GitHub Pages (generated by `npm run build`)

## Key Workflows

### Retrain the model (minor version bump)
```bash
python3 train.py
```
Auto-increments the minor version (1.01 → 1.02). Overwrites root `model.joblib` and `dashboard-data.json`. Updates `versions.json` with metrics. No snapshot directory created.

### Major release retrain
```bash
python3 train.py --major --notes "Backfilled audio features"
```
Bumps to next major version (1.xx → 2.00). Creates a `versions/v2.00/` snapshot with frozen dashboard-data.json, model.joblib, and training-data.md. Use for data backfills, new attributes, or schema changes.

### Compare model versions
Open `compare.html` in a browser. Select two versions to see side-by-side metrics, feature importance, and prediction differences. The version timeline shows R² and RMSE across all versions.

### Score a candidate track
```bash
python3 predict.py --json '{"title":"Track","artist":"Artist","energy":5,...}'
```
See `predict.py` header for full usage. The JSON must include all fields the model expects (energy, tempo, moods, subgenres, instrumentation, audio_features, etc).

### View the dashboard
Open `index.html` in a browser (or serve locally with `python3 -m http.server`). It fetches `dashboard-data.json` from the same directory.

## Feature Engineering

The model uses ~140 features across these categories:
- **Core numeric:** energy, year, tempo, harmonic_complexity
- **Era:** one-hot encoded + decade buckets
- **Instrument:** primary instrument group one-hot
- **Moods:** common moods one-hot + mood_count + VAD axes (see below)
- **Subgenres:** common subgenres one-hot + subgenre_count
- **Ensemble:** ensemble_size, has_guitar, has_strings, has_vocals, is_pianoless, has_trombone, has_horn_section, instrumentation_diversity
- **Label:** common labels one-hot
- **Key players:** key_player_count, artist_is_leader, collaborator_quality (LOO)
- **Discovery:** source one-hot (self, claude-recommendation, autoplay)
- **Interactions:** energy × tempo, energy × complexity, valence × energy, valence × arousal, arousal × energy
- **Artist:** artist_mean_rating (LOO), artist_track_count, artist_consistency, artist_is_new
- **Audio:** duration_s, popularity, tempo_bpm, time_signature, is_live, is_minor_key, is_dorian, duration buckets
- **Format:** is_electric, is_solo_piano, is_collaboration

Dataset is small (~95 tracks), so avoid features that add noise without signal. LOO encoding is used for anything derived from ratings.

### Mood Axes (VAD)

Instead of binary positive/negative mood classification, the model uses a three-dimensional emotion space based on the **circumplex model of affect**:

- **Valence** (-1 to +1): sad/dark ←→ cheerful/bright
- **Arousal** (-1 to +1): calm/subdued ←→ intense/energetic
- **Dominance** (-1 to +1): gentle/receptive ←→ powerful/commanding

The mapping lives in `mood-axes.json` — each of the ~87 mood chips is scored on all three axes from a **music context** perspective (not general English). For example, "bluesy" has negative valence in standard lexicons but is scored as only mildly negative here since it's a positive aesthetic in jazz.

When a track has multiple mood tags, the model averages each axis across all tags to produce three continuous features: `avg_valence`, `avg_arousal`, `avg_dominance`. These replace the old `mood_polarity`, `mood_density_ratio`, and `has_negative_mood` binary features.

**To add a new mood chip:** Add its VAD scores to `mood-axes.json`, add the chip to the Review page's `MOOD_SECTIONS` array, and retrain.

## Dashboard

The dashboard is a React + Vite SPA using Chart.js with a dark theme. Pages:
- **Dashboard** (`/`) — Model Performance, Taste Clusters, Feature Importance, Predicted vs Actual, Rating Distribution, Cluster Profiles, Correlations, Constellation, Circle of Fifths, Taste vs World, Artist Journeys, Mood × Rating, Mood Emotion Space (VAD axes), Replayability, Year/Duration Rating, Instrument/Label Breakdown, Biggest Misses, Top Artists, Era Breakdown, Harmonic Complexity, Discovery Source, Sound Profile
- **Wrapped** (`/wrapped`) — Cover Flow carousel (CSS scroll-driven animations) showing Top Tracks, Top Artists, Top Albums, Albums by Liked. Uses `training-data.json` and `artwork-cache.json`. Filter buttons for era, mood, instrument, liked status. Artists/albums require 2+ tracks reviewed.
- **Compare** (`/compare`), **Review** (`/review`), **Dictionary** (`/dictionary`), **Playlists** (`/playlists`)

When adding visualizations:
- Match the dark-theme CSS variables (see `:root` block)
- Use `panel span-6` or `panel span-12` grid layout
- Add a nav link in the `<nav class="section-nav">` block
- Wire the render function in the fetch chain

### Build & Deploy
The site deploys to GitHub Pages from the `docs/` directory. **After any frontend change, you must build and push the production assets:**
```bash
cd ~/jazz-ml && npm run build && git add docs/ && git commit -m "Build production assets" && git push
```
The build script (`npm run build`) runs `vite build` and copies `dashboard-data.json`, `versions.json`, `training-data.json`, and `artwork-cache.json` into `docs/`. Pushing source files alone will NOT update the live site.

### Artwork Cache
`artwork-cache.json` maps album and artist names to iTunes artwork URLs. When new albums are added to `training-data.json`, refresh the cache:
```bash
cd ~/jazz-ml && python3 fetch-artwork.py
```
